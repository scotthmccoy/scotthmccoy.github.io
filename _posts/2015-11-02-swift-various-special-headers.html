  1. ProductModuleName-Bridging-Header.h: Import every Objective-C header I want to expose to Swift. Make sure the "Objective-C Bridging Header" build setting has a path to the file (it'll halt compilation if it doesn't)
  2. Foo-Swift.h: This is the opposite of the bridging header. It exposes Swift classes to Obj-c that are either marked @objc or are declared as subclasses of ObjC classes (frequently NSObject). This header is generated by Xcode, you don't need to generate it or maintain yourself. It does sometimes seem like it can get out of sync, though. Usually a clean re-build fixes this.
  3. FrameworkName.h: This is an umbrella header for your framework. Inside you should import all the public headers of your framework using statements like #import . I think you may also have to add something like the following, but I am not sure:
    1. //! Project version number for OpenXSDKCore.
    2. FOUNDATION_EXPORT double OpenXSDKCoreVersionNumber;
    3. //! Project version string for OpenXSDKCore.
    4. FOUNDATION_EXPORT const unsigned char OpenXSDKCoreVersionString[];
